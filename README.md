# Musik-Quiz
Gruppenname: 

1. Two-Sided Platform:
Seite A: Der „Quiz-Master“ (Creator)
Aktion: Erstellt eigene Quiz-Pakete. Er sucht über die Spotify-API nach Songs, stellt ein Set zusammen (z.B. „Best of 90s Techno“) und gibt dem Paket einen Namen.
Ziel: Content erstellen und sehen, wie viele Leute sein Quiz erfolgreich abgeschlossen haben.

Seite B: Der „Herausforderer“ (Player)
Aktion: Durchsucht die Plattform nach Quiz-Paketen anderer Nutzer und spielt diese.
Ziel: Highscores knacken und sich in einer Rangliste (Leaderboard) verewigen.

2. Anpassung der „Value Proposition“
Target User:
- Musik-Nerd-Communities, die ihr Wissen in Nischen-Genres testen wollen
- Nutzung als Spiel für Parties oder ähnliches
Plausible Problem: Bestehende Musik-Quizze sind oft zu oberflächlich oder decken spezielle Musikrichtungen (z.B. „Indie-Rock aus Berlin-Spandau“) nicht ab.
Solution: Eine Plattform, auf der Experten (Quiz-Master) maßgeschneiderte Audio-Challenges erstellen können, die von der Community (Player) gelöst werden. 

4. Target Scope: Die UI-Screens (Deine Scribbles)
Du musst für die Abgabe Skizzen erstellen. Plane mindestens diese 4 Ansichten ein:
1. Home/Discovery: Liste der neuesten oder beliebtesten Quiz-Pakete (erstellt von Usern).
2. Suchfunktion mit kategorien: Eine Funktion zur Suche spezifischer Quizze durch Schlüsselworte oder Kategorisierung der Sucheinstellungen
3. Quiz-Creator Dashboard: Eine Suchmaske (Spotify API Anbindung), um Songs zu einem eigenen Paket hinzuzufügen.
4. Game Interface: Der Player mit dem 30-Sekunden-Snippet, dem Eingabefeld und dem Timer.
5. Leaderboard: Eine Übersicht, welcher User welches Quiz am schnellsten gelöst hat.

6. Umsetzung der „Team Composition“ & „Meta-Goals“
Team Name: „Music competitors“

Meta-Goals:
Target Grade: 1.7
Personal Goals: Sicherer Umgang mit APIs(OAuth2 Spotify Login), Umsetzung einer App mit Frontend und Backend, Sicherer Umgang mit Creator-views
